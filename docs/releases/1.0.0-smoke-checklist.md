## 1) Root-safe read-only stays in repo root

Action:
- Run a clearly read-only flow such as `read`, `glob`, `grep`, or read-only `git status`.

Observe:
- [ ] No new worktree is created for the read-only flow.
- [ ] No active task binding is created just to satisfy the read-only flow.

## 2) First mutating call provisions and binds a worktree

Action:
- Trigger one small mutating action such as `apply_patch`, `edit`, or a mutating `bash` command.

Observe:
- [ ] A worktree is provisioned or selected before the mutating action runs.
- [ ] The mutating action runs in the bound worktree, not repo root.

## 3) Delegation binds before execution and is reused across one linear flow

Action:

Observe:
- [ ] The binding exists before delegated execution starts.
- [ ] The same `task_id` and `worktree_path` are reused across the linear flow.

## 4) Path and workdir rewriting behave safely

Action:
- Use one repo-scoped path or workdir that should be rewritten into the active worktree.
- Use one clearly external absolute path that should stay untouched.
- If feasible, try one ambiguous path case that should block rather than guess.

Observe:
- [ ] Repo-scoped paths/workdirs are rewritten into the bound worktree.
- [ ] External absolute paths are not rewritten into the worktree.
- [ ] Ambiguous or opaque repo-root references block execution instead of guessing.

- Notes/output:

## How to record results

- Create one GitHub issue per alpha named `1.0.0-alpha.N smoke test results`.
- Paste this checklist into the issue and fill in the evidence fields.
- If a check fails, open a separate bug issue, link it from the smoke-test issue, and add it to milestone `1.0.0` if it blocks the stable release.
